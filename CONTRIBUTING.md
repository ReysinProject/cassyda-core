# Contributing to ReysinProject Libraries

Thank you for your interest in contributing to a ReysinProject! This document provides guidelines and instructions for contributing.

## Code of Conduct

This project adheres to a Code of Conduct that all contributors are expected to follow. Please read [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) before contributing.

## How to Contribute

### Reporting Issues

- Check if the issue already exists
- Use the issue template
- Provide clear steps to reproduce
- Include relevant information:
    - Library version
    - Environment details
    - Error messages
    - Code samples

### Pull Requests

1. Fork the repository
2. Create a new branch
3. Make your changes
4. Write/update tests
5. Update documentation
6. Submit PR using template

### Development Setup

1. Clone the repository:
```bash
git clone git@github.com:ReysinProject/cassyda-core.git
```

2. Install dependencies:
```bash
npm install
```

3. Run tests:
```bash
npm test
```

### Coding Standards

- Follow TypeScript best practices
- Use ESLint configuration
- Write JSDoc comments
- Include unit tests
- Keep PR scope focused

### Testing

- Write unit tests for new features
- Maintain existing test coverage
- Run full test suite before submitting PR

### Documentation

- Update README.md if needed
- Add JSDoc comments
- Update API documentation
- Include examples for new features

## Release Process

1. Version bump
2. Update CHANGELOG.md
3. Create release PR
4. Tag release
5. Deploy to npm

## Questions?

Feel free to ask in GitHub Issues or Discussions.
